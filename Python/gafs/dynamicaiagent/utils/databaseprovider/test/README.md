# DatabaseProvider — Test Suite

This directory contains integration tests for the `databaseprovider` component.
All tests use **real database connections** — there are no mocks.

## Files

| File | Description |
|---|---|
| `test_surrealdb_local_provider.py` | Tests for `SurrealDbLocalProvider` (embedded SurrealDB) |
| `test_surrealdb_remote_provider.py` | Tests for `SurrealDbRemoteProvider` (remote SurrealDB over WebSocket) |
| `test_build_databaseprovider.py` | Build test — runs local provider tests against the compiled `.pyd` |
| `test_build_remote_provider.py` | Build test — runs remote provider tests against the compiled `.pyd` |
| `data_models.py` | Shared `TestRecord` data class used by both test suites |
| `secret_test_config_surrealdb_local.json` | Connection config for local (embedded) tests |
| `secret_test_config_surrealdb_remote.json` | Connection credentials for remote tests (**do not commit**) |

## Configuration

### `secret_test_config_surrealdb_local.json`

```json
{
  "namespace": "dynamic-ai-agent",
  "database": "dynamic-ai-agent",
  "storage_type": "mem"
}
```

`storage_type` may be `"mem"`, `"surrealkv"`, `"rocksdb"`, or `"file"`.
For CI environments, use `"mem"` (no filesystem side-effects).

### `secret_test_config_surrealdb_remote.json`

```json
{
  "endpoint": "wss://<host>/rpc",
  "namespace": "<namespace>",
  "database": "<database>",
  "username": "<username>",
  "password": "<password>",
  "auth_level": "namespace"
}
```

`auth_level` may be `"root"`, `"namespace"`, or `"database"`.

> **Important**: This file contains credentials. It must **not** be committed to
> version control. Add it to `.gitignore` and distribute it via a secure secret
> manager.

## Running the Tests

All commands must be run from the `Python/` project root.

```bash
# Local provider tests (fast — uses in-memory embedded SurrealDB)
pytest gafs/dynamicaiagent/utils/databaseprovider/test/test_surrealdb_local_provider.py -v

# Remote provider tests (requires network + remote SurrealDB instance)
pytest gafs/dynamicaiagent/utils/databaseprovider/test/test_surrealdb_remote_provider.py -v

# Both source tests together
pytest gafs/dynamicaiagent/utils/databaseprovider/test/ -v \
  --ignore=gafs/dynamicaiagent/utils/databaseprovider/test/test_build_databaseprovider.py \
  --ignore=gafs/dynamicaiagent/utils/databaseprovider/test/test_build_remote_provider.py

# Build tests (requires compiled .pyd — run build_nuitka.py first)
pytest gafs/dynamicaiagent/utils/databaseprovider/test/test_build_databaseprovider.py -v
pytest gafs/dynamicaiagent/utils/databaseprovider/test/test_build_remote_provider.py -v
```

## Test Structure

### `test_surrealdb_local_provider.py`

| Class | Count | Description |
|---|---|---|
| `TestLocalSurrealDbOptions` | 8 | Options validation, `to_dict`, `from_dict`, equality |
| `TestRetryOptions` | 5 | RetryOptions value-object semantics |
| `TestInitialize` | 5 | Provider initialization (valid, invalid options, storage types) |
| `TestQueryRaw` | 6 | `query_raw()` — results, errors, retry, not-initialized |
| `TestQuery` | 8 | `query()` — typed deserialization (many, single, none, no model) |
| `TestClose` | 2 | `close()`, async context manager |

### `test_surrealdb_remote_provider.py`

| Class | Count | Description |
|---|---|---|
| `TestRemoteSurrealDbOptions` | 5 | Options validation, `to_dict`, `from_dict` |
| `TestInitialize` | 5 | Provider initialization (valid creds, wrong password, wrong namespace) |
| `TestQueryRaw` | 5 | `query_raw()` — results, errors, not-initialized |
| `TestQuery` | 8 | `query()` — typed deserialization |
| `TestClose` | 2 | `close()`, async context manager |

## Prerequisites

- SurrealDB Python SDK installed (`pip install surrealdb`)
- For remote tests: a reachable SurrealDB Cloud or self-hosted instance
- For build tests: compiled `.pyd` file at
  `Python/build/<arch>/gafs/dynamicaiagent/utils/databaseprovider.cp312-win_amd64.pyd`
  (generated by `gafs/dynamicaiagent/utils/databaseprovider/build_nuitka.py`)
